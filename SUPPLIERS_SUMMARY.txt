🔄 COMPLETE WORKFLOWS:

   Workflow 1: Create Supplier
   ────────────────────────────────────────────────────────────
   1. Navigate to /suppliers
   2. Click "New Supplier"
   3. Fill in form (name required)
   4. Click "Create Supplier"
   5. Success toast shown
   6. Redirected to /suppliers
   7. New supplier visible in list

   Workflow 2: Edit Supplier
   ────────────────────────────────────────────────────────────
   1. Find supplier, click Edit icon
   2. Modify fields
   3. Click "Update Supplier"
   4. Success toast shown
   5. Changes reflected in list

   Workflow 3: Delete Supplier (No POs)
   ────────────────────────────────────────────────────────────
   1. Find supplier, click Delete icon
   2. Confirm deletion
   3. Supplier deleted
   4. Success toast shown
   5. List refreshed

   Workflow 4: Delete Supplier (With POs)
   ────────────────────────────────────────────────────────────
   1. Find supplier with POs, click Delete
   2. Confirm deletion
   3. Error toast: "Cannot delete supplier with existing purchase orders"
   4. Supplier NOT deleted
   5. User can view supplier detail to see linked POs

   Workflow 5: Create PO with Existing Supplier
   ────────────────────────────────────────────────────────────
   1. Go to /purchase-orders/new
   2. Select supplier from dropdown
   3. Add products
   4. Save PO
   5. PO created with supplier_id
   6. Supplier name clickable in PO detail

   Workflow 6: Create PO with Quick Add Supplier
   ────────────────────────────────────────────────────────────
   1. Go to /purchase-orders/new
   2. Click "+" button next to supplier dropdown
   3. Modal opens
   4. Fill: Name, Phone, Email
   5. Click "Create Supplier"
   6. Supplier created
   7. Dropdown auto-selects new supplier
   8. Modal closes
   9. Continue with PO creation
   10. Save PO

   Workflow 7: Create PO with Manual Entry
   ────────────────────────────────────────────────────────────
   1. Go to /purchase-orders/new
   2. Type supplier name in "Or Enter Supplier Name"
   3. Add products
   4. Save PO
   5. PO created with supplier_name (no supplier_id)
   6. Supplier name shown in PO detail (not clickable)

   Workflow 8: Navigate from PO to Supplier
   ────────────────────────────────────────────────────────────
   1. Go to /purchase-orders
   2. Click View on PO
   3. Click supplier name
   4. Navigate to /suppliers/:id
   5. See supplier details
   6. See all POs from this supplier

🔍 DATABASE SCHEMA:

   suppliers table:
   ────────────────────────────────────────────────────────────
   • id (uuid, primary key)
   • name (text, required)
   • contact_person (text, nullable)
   • phone (text, nullable)
   • email (text, nullable)
   • address (text, nullable)
   • note (text, nullable)
   • status (text, default 'active', check: active/inactive)
   • created_at (timestamptz, default now())
   • updated_at (timestamptz, auto-updated)

   Indexes:
   ────────────────────────────────────────────────────────────
   • idx_suppliers_name (name)
   • idx_suppliers_status (status)
   • idx_suppliers_email (email)
   • idx_suppliers_phone (phone)

   RLS Policies:
   ────────────────────────────────────────────────────────────
   • SELECT: authenticated users
   • INSERT: authenticated users
   • UPDATE: authenticated users
   • DELETE: authenticated users (with PO protection in API)
